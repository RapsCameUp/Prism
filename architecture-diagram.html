Data Flow Pipelines

PRIMARY Splunk → Incident Detection & Telemetry

  1. 1CSV data uploaded to 4 Splunk indexes (incidents, main, deployments, metrics)
  2. 2SplunkService sends SPL queries via REST API POST to /services/search/jobs/export
  3. 3Parses newline-delimited JSON response (extracted fields or _raw CSV fallback)
  4. 4Incident Service merges Splunk-detected + DB-predicted incidents
  5. 5Telemetry Agent queries error logs & patterns during investigation
  6. 6Deployment Agent fetches CI/CD events for correlation analysis

PREDICTION Splunk Metrics → CDTSM → Predicted Incidents

  1. 1Scheduler triggers Predictive Agent every 2 minutes
  2. 2Agent fetches 5 metrics per service from Splunk (memory, CPU, latency, error rate, request rate)
  3. 3Splits time series into coarse_ctx (trend) + fine_ctx (recent high-res)
  4. 4POST to CDTSM /cdtsm/v1/ai/infer with horizon=6, quantiles=[p10,p50,p90]
  5. 5Computes anomaly score: trend_acceleration × 50 + p90_divergence × 30 + 20
  6. 6If score ≥ 70: creates predicted incident (source: cdtsm-prediction)

AI Gemini AI — Multi-Agent Investigation

  1. 1Coordinator dispatches 5 specialized agents sequentially
  2. 2Telemetry Agent sends Splunk error patterns → Gemini identifies anomalies
  3. 3Deployment Agent sends timeline → Gemini calculates correlation score
  4. 4Root Cause Agent synthesizes all findings → Gemini produces diagnosis
  5. 5Remediation Agent → Gemini recommends prioritized actions
  6. 6Ask AI endpoint provides contextual Q&A with full incident context

AUTOMATION GitHub — PR Traceback & Auto-Remediation

  1. 1PR Traceback Agent fetches suspected PR metadata + diff from GitHub
  2. 2Gemini analyzes code changes to determine risk score
  3. 3Investigation Service auto-creates GitHub Issue for critical incidents
  4. 4Remediation Service: create branch → commit fix → open PR
  5. 5Assigns reviewers (repo owner + rrabelani7)
  6. 6Human engineer reviews and approves (never auto-merges)
